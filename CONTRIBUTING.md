# Contributing to Compiler Castle

Thank you for your interest in contributing to Compiler Castle! We want to make contributing to this project as easy and transparent as possible.

## How to Contribute

### Reporting Bugs

Open a GitHub issue with a clear title and description. Include code snippets that reproduce the issue if possible.

### Suggesting Enhancements

We love new ideas! Open an issue to discuss your suggestion before starting work on it.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style.
5. Issue that pull request!

## Development Setup

1. Clone the repository.
2. Open in IntelliJ IDEA or VS Code.
3. Run `./gradlew build` to ensure everything is working.

## Code Style

We follow the standard Kotlin coding conventions.
